BillPouch

A P2P social distributed filesystem — written in Rust.

MIT / Apache-2.0 Rust 2021 libp2p 0.54 Alpha
Get Started API Docs GitHub

Overview

A leaderless filesystem where storage is social

BillPouch is a fully distributed filesystem built on libp2p. There is no central server and no coordinator — every node is equal. Participants contribute disk space, and the network tracks who contributes what using a gossip-based DHT. Your Ed25519 keypair is your identity across every network you join.

The name comes from the pelican: a bird that hunts, stores, and shares. Three service types map directly onto how a pelican operates.

Service Pelican analogy Role in the network
pouch The throat sac Bids a portion of local storage into the network. Files are stored here.
bill The beak Personal file I/O interface — your window into files spread across pouch nodes.
post The wings Pure routing / relay — contributes only CPU and bandwidth; no storage required.
⚙️

Truly P2P

No central server. Kademlia DHT and gossipsub keep peers discovered and connected.

🔒

Encrypted Transport

All connections use the Noise protocol for authentication and encryption, multiplexed with Yamux.

👤

Ed25519 Identity

Your keypair is your user ID. Multiple nodes can share one identity across any number of networks.

🌐

Multiple Networks

One daemon instance participates in several independent networks simultaneously.

🔌

Zero-Config LAN Discovery

Built-in mDNS finds peers on the same local network with no configuration needed.

🔧

Modular Core

bp-core is a pure Rust library. REST, gRPC, or WASM adapters can be layered on top without touching core logic.

Architecture

How it fits together

BillPouch is a Cargo workspace with two crates: bp-core (pure library, zero I/O) and bp-cli (the bp binary). The CLI spawns a background daemon (bpd) on first use and communicates with it over a Unix domain socket.

IPC: CLI to Daemon

Every bp subcommand is a thin client. It connects to the bpd daemon via a Unix socket and exchanges newline-delimited JSON messages. The daemon owns the libp2p swarm and the service registry; the CLI never touches the network stack directly.

The CLI is a thin JSON client. All state and the full libp2p swarm live inside bpd.

NodeInfo Gossip Protocol

Every node periodically broadcasts a NodeInfo message on the gossipsub topic billpouch/v1/{network_id}/nodes. The metadata field is an open map — future extensions (storage quotas, versions, capabilities) are added there without breaking existing nodes.

sequenceDiagram participant A as Node A participant T as gossipsub topic
billpouch/v1/{net}/nodes participant B as Node B participant C as Node C A->>T: publish NodeInfo T-->>B: deliver NodeInfo(A) T-->>C: deliver NodeInfo(A) B->>B: update local DHT state C->>C: update local DHT state Note over A,C: Periodic heartbeat — all nodes re-announce themselves

Gossip propagates peer presence; the intentionally open metadata field allows schema evolution.

Quick Start

Up and running in minutes

You only need Rust 1.75+. No other system dependencies are required.

1

Clone and build

Compile a release binary or install it globally with Cargo.

git clone https://github.com/Onheiron/BillPouch.git cd BillPouch cargo build --release # Binary is at ./target/release/bp # …or install globally: cargo install --path crates/bp-cli
2

Create your identity

Generates an Ed25519 keypair at ~/.local/share/billpouch/identity.key. Your fingerprint (first 8 bytes of your public key SHA-256) is your permanent user ID across every network.

bp login --alias "your-name"
3

Hatch a service

The first hatch automatically starts the background daemon. Each service is assigned a unique UUID printed on startup.

# Bid 10 GiB of local storage into a network bp hatch pouch --network my-network --storage-bytes 10737418240 # Start a personal file I/O interface bp hatch bill --network my-network # Start a relay-only node (no storage required) bp hatch post --network my-network
4

Inspect the flock

List your local services, joined networks, and all known peers in a formatted view.

bp flock
5

Stop a service or log out

Farewell a specific service by UUID, or remove your identity key entirely (irreversible — back it up first).

bp farewell a3f19c2b-e9d2-4f1a-bc30-112233445566 # Remove identity key — back up identity.key first! bp logout
